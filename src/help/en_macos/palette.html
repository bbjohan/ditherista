Palette Settings

The Palette tab allows defining color palettes for both Mono and Color dithering.

Mono Palette Settings

In the Mono Palette section you can pick different colors than just black and white for mono dithering. Pressing the X button resets the colors to black and white.

Color Palette Settings

The palette and the color comparison method play a very important role when it comes to the quality of color dithering. The better the target palette matches the different colors and brightness of the source image, the better the dithering result will be.

Palette source - lets you pick the target palette the dithered image should use:

  • built-in: allows you to choose a pre-defined palette, such as used in classic computer systems and video game consoles.
  • from file: allows you to load a Paint.NET compatible palette. You can get such palettes from lospec.com or create your own using a text editor.
  • reduced: creates a palette with a limited amount of colors from the current image.
  • custom: this entry will become visible when a color of the target palette has been edited.
Color comparison - lets you pick a method to match the source image's original pixel colors to the next closest color in the target palette (as chosen in Palette source):
  • LAB '76: A fairly fast method using the LAB color space. Sometimes the results are good but for best quality pick LAB '94 or LAB 2000.
  • LAB '94: A method offering a good balance between speed and quality, using the LAB color space.
  • LAB 2000: Offers the best quality in situations where the reduced palette matches the original palette well. Can be slow.
  • sRGB: The best all-around color matching method, using the sRGB color space.
  • sRGB CCIR: Same as sRGB. Takes human perception into account. Sometimes better than sRGB.
  • HSV: Uses the HSV color space. May not work properly with small palettes where the colors do not match the source image well.
  • Luminance: A great choice for uniform color palettes with a color gradients. Color matching is less accurate than in other methods, but it get very pretty looking results with a gradient palette.
  • Linear: Compares colors in linear space; generally not recommended, but may work well for palettes which have few colors and do not match the source image's palette well.
  • Linear CCIR: Same as Linear but takes human perception into account.
  • Tetrapal: A method intended for ordered dithering, but it also works for other dithering approaches.

Loading Palettes from File

When the Palette source is set to from file, you can press the Browse button and load a Paint.NET compatible palette file. You can get such palettes from lospec.com or create your own using a text editor. You can also type or paste the path to a Paint.NET palette file into the text field next to the Browse button.

Creating a Reduced Palette

When the Palette source is set to reduced you can specify the number of colors your target palette should have into the input field next to Colors. You can then choose a Color reduction mode:
  • Median Cut: The best all-around method. It's fast and generally quite accurate.
  • Wu: Often gives better results than Median Cut, but not always. It's also slower.
  • KD-Tree: Generally gives better results than Median Cut. It's the slowest method.
In some cases, for example when the number of target colors is low, the target palettes colors may not be well distributed. In this case you can turn on Count unique pixels. This will not just take into account the overall number of unique colors in the source image, but also how many pixels there are of each color. Note that this makes the color reduction slower!

In some cases, for example when the number of target colors is low, it can help to explicitly include extreme colors, e.g. very dark, very bright, very blue, very red, etc. colors:

  • Include brightest / darkest: includes the 2 colors from the source image that are closest to pure white and pure black, if there is space in the target palette.
  • Include max RGB colors: includes the 3 colors from the source image that are closest to pure red, green and blue, if there is space in the target palette.
  • Include max CMY colors: includes the 3 colors from the source image that are closest to pure cyan, magenta and yellow, if there is space in the target palette.

Color Swatches and Palette Editing

The color swatches are a powerful way to preview and edit the colors of a palette used for dithering. You can edit the swatches the following ways:
  • Double-click on a swatch or onto a hex color label to bring up a color picker dialog. You can also open the dialog by right-clicking onto a swatch and choosing edit color.
  • Right-click on a swatch and choose copy color to copy a color's hex value to the clipboard. The color data in the clipboard is just a piece of text, e.g. "#FFFF0000 for red.
  • Right-click and choose paste color to copy a color's hex value from the clipboard and to change the color. You are not limited to copy colors within Ditherista. You can copy color values from e.g. a text editor too, and paste them into Ditherista. You can also copy colors between the Color Palette and Mono Palette sections.
When edited, a new entry named custom will appear in the Palette source drop down.

The hex color values used consist 4 channels: alpha (i.e. transparency), red, green and blue.

Custom Palette Export

When you edit colors of a palette, the resulting new palette will be available under the custom entry in the Palette source drop down.

There is also a button save palette which lets you save your custom palette as Paint.NET file.
